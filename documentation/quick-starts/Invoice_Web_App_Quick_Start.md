# Invoice Upload Web App — Quick Start Guide

## What This Tool Does

Upload a PDF invoice, check that the data looks right, and click Save — it goes straight into Airtable. No spreadsheets, no typing.

---

## How to Access It

Go to: **https://invoicewebapp-navy.vercel.app**

Works on any browser (Chrome, Firefox, Edge, Safari). Works on your phone too.

No login needed.

---

## How to Use It

### Step 1: Upload your invoice

Click the upload area (or drag your PDF file onto it). You'll see the file name appear once it's loaded.

### Step 2: Click "Process Invoice"

The tool reads the PDF and pulls out the key information automatically. This takes about 5–10 seconds.

### Step 3: Review the fields

You'll see the extracted data in editable boxes:
- Invoice Number
- Vendor
- Invoice Date
- Due Date
- Amount

**Check that everything looks right.** If anything is wrong or missing, just click the field and type the correct value.

### Step 4: Click "Save to Airtable"

That's it. The record is saved. You'll see a confirmation message.

---

## What the Confidence Badge Means

After processing, you'll see a percentage badge (like "85% confidence"). This tells you how confident the tool is in the extraction.

- **80–100%** — Data looks good, double-check quickly
- **Below 80%** — One or more fields may need correction, review carefully

Either way, you can always edit the fields before saving.

---

## What If Something Goes Wrong?

**Problem:** The page won't load
**Solution:** Try refreshing your browser (press F5). If it still won't load, wait 5 minutes and try again.

**Problem:** I uploaded a file but nothing happened
**Solution:** Make sure it's a PDF file (not a photo or Word document). Try again.

**Problem:** The data looks completely wrong
**Solution:** Manually type the correct values into the fields and click Save. Then let your IT contact know which invoice format caused the issue.

**Problem:** "Save to Airtable" button shows an error
**Solution:** Contact IT — this usually means a credentials issue that needs to be fixed on the backend.

**Problem:** I saved a record with wrong data
**Solution:** Go directly to Airtable and edit the record there. The tool doesn't prevent manual edits in Airtable.
